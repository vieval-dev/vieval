# 为什么选择 Vieval

评测代码需要随被测行为一起演进。Vieval 将评测文件放在产品代码旁，使两者能够通过同一套开发流程接受评审、进入版本控制并持续修改。

## 可重复执行不只需要一个回调函数

回调函数可以检查一次行为。要让评测稳定复现，还需要找到预期文件，以一致的方式展开变量组合，调度得到的工作，并在执行后保留证据。

Vieval 负责这段评测生命周期：发现评测、展开矩阵、调度、记录执行证据、生成报告与比较结果。任务仍然由普通 TypeScript 代码定义；运行器补齐周边结构，使同一个评测可以再次执行，也可以按多个模型和场景组合执行。

## 一次执行同时服务于人和工具

评测执行期间，CLI 提供便于阅读的进度与摘要。传入 `--json` 可以向标准输出打印机器可读的结果；传入 `--report-out <directory>` 才会写入供分析、比较和自动化流程检查的报告制品。这些输出模式与交互式 CLI 复用同一套发现、调度和执行流程。

## 适合的场景

以下需求适合使用 Vieval：

- 评测属于 TypeScript 仓库，并且需要与产品改动一起评审；
- 同一个任务需要按模型、场景、评分标准或数据集变量执行；
- 本地执行与自动化流程需要一致的调度方式和执行证据；
- 用例级断言、分数与指标需要通过显式写入的报告制品保留下来。

## 不适合的场景

Vieval 不提供托管式数据集管理、数据标注界面或标注产品，也不提供 SaaS 可观测性服务。这些需求应交给对应的专用服务。Vieval 可以评测读取外部数据或调用远程模型的代码，但它的职责仍是运行评测和生成制品，而不是托管这些周边产品。
