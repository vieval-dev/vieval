# 核心概念

Vieval 用一组相互嵌套的概念组织评测。指南中的各项能力都可以放回下面这段生命周期理解：

```text
项目
  -> 任务 <-> 运行矩阵 + 评测矩阵
    -> 用例 -> 断言 / 分数 / 指标证据

任务 -> 尝试 -> 重试

执行 --json -> 机器可读的标准输出
执行 --report-out -> 报告制品 -> 分析 / 比较
```

**项目**定义发现边界和项目级配置。被发现的 eval 定义可以携带一个**任务**，而 `describeTask` 等 DSL 定义会直接注册任务。矩阵中的各行再把已注册的任务展开成具体的调度项。一个任务包含一个或多个**用例**；断言、归一化分数和自定义指标会在用例中形成执行证据。

可靠执行分为两个范围。一次**尝试**会完整重跑任务，并保留每次尝试产生的证据。一次**重试**则让失败的用例在同一次尝试内重新运行。输出方式需要显式指定：`--json` 向标准输出打印机器可读的结果，`--report-out <directory>` 写入供分析和比较命令读取的报告制品。

| 概念 | 职责 | 常见范围 |
| --- | --- | --- |
| 项目 | 定义发现规则、配置、执行器与矩阵默认值 | 一个产品领域或一组评测 |
| 任务 | 描述评测行为，并连接运行矩阵与评测矩阵的选择结果 | 一项按多个变量执行的被测行为 |
| 用例 | 执行一个输入或场景，并产生断言、分数和指标证据 | 任务中的一次检查 |
| 运行矩阵 | 展开模型、场景等执行侧变量 | 项目、评测定义或任务定义 |
| 评测矩阵 | 展开评分标准等评测侧变量 | 项目、评测定义或任务定义 |
| 尝试 | 完整重复任务，用于衡量可靠性 | 一个已调度任务 |
| 重试 | 在一次尝试内重新执行失败的用例 | 一次用例执行 |
| 报告制品 | 使用 `--report-out` 时保存执行摘要、事件与用例证据 | 一次执行及其配置的报告目录 |
| 分析与比较 | 读取制品以检查结果，或比较多次执行和多种方法 | 一组或多组报告制品 |
